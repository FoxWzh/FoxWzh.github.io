> 一句话摘要：量化是给模型"瘦身"，蒸馏是让小模型"拜名师"——两者加在一起，让 7B 的模型能打出以前 70B 才有的效果，读完你能判断什么场景该用小模型代替大模型。

---

## 是什么

### 量化（Quantization）：给模型压缩体积

大模型里有海量的数字（参数权重），默认用 **FP16（16位浮点数）** 存储，精度高但占空间大。

**量化**就是把这些数字换成更低精度的格式，比如 INT8（8位整数）或 INT4（4位整数）——就像把一张 4K 照片压成 1080p，文件小了很多，但肉眼看差别不大。

```
FP16（原始精度）：每个参数占 2 字节
  → 7B 模型需要约 14GB 显存

INT8（Q8）：每个参数占 1 字节
  → 7B 模型需要约 7GB 显存（省一半）

INT4（Q4）：每个参数占 0.5 字节
  → 7B 模型需要约 3.5GB 显存（省75%）
```

### 蒸馏（Distillation）：让小模型拜大模型为师

**知识蒸馏**的思路是：用一个大模型（Teacher，老师）的输出来训练小模型（Student，学生），让小模型学会大模型的"思维方式"，而不只是死记硬背答案。

类比：一个博士生（大模型）把自己的知识浓缩成一本精华教材，一个高中生（小模型）读这本教材，能比直接啃论文学得快、学得好。

---

## 为什么重要

### 场景 1：用户要求数据不出本地

医疗、法律、金融等对数据隐私要求极高的场景，不能把数据发到云端 API。量化后的小模型可以跑在用户本地设备上，数据完全不出境。

### 场景 2：API 成本压不下来

GPT-4o 级别的模型调用成本高，但很多简单任务（分类、摘要、格式转换）其实用小模型就够了。把这部分请求路由到量化后的小模型，成本能降低 90% 以上。

### 场景 3：移动端/边缘设备 AI 功能

手机、平板、IoT 设备的算力和内存有限，只有量化到 Q4 甚至更低精度的模型才能跑起来。

## 核心内容

### Q8 vs Q4：怎么选？

| | Q8（INT8） | Q4（INT4） |
|--|-----------|-----------|
| 精度损失 | 极小，几乎无感 | 轻微，大多数任务不影响 |
| 显存节省 | 50%（对比 FP16） | 75%（对比 FP16） |
| 推理速度 | 比 FP16 快 | 比 Q8 更快 |
| 适用场景 | 对准确率敏感的任务（代码、数学） | 对话、摘要、翻译等通用任务 |
| 硬件要求 | 中等 GPU 即可 | 笔记本 CPU 也能跑 |

**2024 年的主流选择**：Q4 是使用最广泛的格式，尤其是 **Q4_K_M**（K-Quant 系列）——它对模型中"重要层"保留更高精度，对"不重要层"压缩更狠，在同等体积下准确率比老式 Q4 高出不少。

---

### 量化的底层原理（PM 需要了解的程度）

量化不是简单地"四舍五入"，核心挑战是：

**权重分布不均匀**：模型里大多数权重数值很小，但偶尔有几个"离群值"非常大。如果用统一的精度量化，这些离群值会拉低整体精度。

现代量化方案（如 GPTQ、AWQ）的解法：
- **分组量化**：把权重分成小组，每组单独计算量化参数
- **重要性加权**：根据权重对模型输出的影响程度，给重要权重分配更高精度

---

### 蒸馏的两种主要形式

**① 输出蒸馏（Response Distillation）**

最简单的形式：用大模型生成大量高质量问答对，再用这些数据训练小模型（SFT）。

```
GPT-4 生成 100 万条高质量对话
      ↓
用这些数据训练 Llama 3-8B
      ↓
小模型学到了 GPT-4 的回答风格和质量
```

这是目前最流行的做法，Alpaca、WizardLM 等知名小模型都是这么做的。

**② 过程蒸馏（Process Distillation）**

更高级的形式：不只学大模型的答案，还学它的"推理过程"。DeepSeek R1 把推理能力蒸馏到 1.5B/7B/14B 小模型就是用这种方式。

```
DeepSeek R1（671B）生成完整思维链
      ↓
用思维链数据训练 Qwen-7B
      ↓
7B 小模型获得了接近大模型的推理能力
```

效果：DeepSeek R1-Distill-7B 在数学推理上的表现超过了很多更大的非推理模型。

### 量化 + 蒸馏的组合拳

实际部署中，两者通常组合使用：

```
原始大模型（如 Llama 3 70B，FP16）
    ↓ 蒸馏
小模型（Llama 3 8B，FP16，能力接近原版 70B）
    ↓ 量化
量化小模型（Llama 3 8B Q4，3.5GB，笔记本可运行）
```

这条路走下来，一个普通消费级笔记本就能跑出两年前需要专业 GPU 服务器才有的能力。

---

## 能力边界 / 局限

- **量化有精度下限**：Q4 以下（如 Q2、Q3）的精度损失开始明显，不适合对准确率敏感的任务。
- **蒸馏不能超越老师**：小模型学大模型，天花板是大模型的能力，不可能通过蒸馏让小模型超越它的老师。
- **知识覆盖面仍有差距**：小模型参数少，能存储的知识量有限，在长尾知识、罕见领域上表现明显弱于大模型。
- **量化对特定任务影响大**：代码生成、数学计算等对精度敏感的任务，Q4 量化带来的精度损失比通用对话更明显。
- **本地部署仍需一定硬件**：即使量化到 Q4，7B 模型还需要约 4-8GB 内存，在低端设备上速度很慢。

## PM 视角的关键结论

1. **小模型 + 量化是降成本的最直接手段**。识别出产品中"不需要大模型能力"的请求，路由到小模型，成本能降 80-95%。

2. **Q4_K_M 是 2024 年本地部署的事实标准**。如果要支持用户本地运行，这个格式的精度/体积比最优，优先推荐。

3. **蒸馏让"够用"的门槛大幅降低**。2024 年的 7B 蒸馏模型，能力已经接近 2023 年的 70B 模型。对于很多垂直场景，不再需要调最贵的 API。

4. **边缘部署是下一个产品机会**。量化 + 小模型让 AI 能真正跑在设备端，这开启了离线使用、隐私保护、低延迟等新的产品可能性。

5. **不要盲目追求小模型**。对于需要深度推理、广泛知识或高准确率的核心功能，大模型仍然是不可替代的。小模型适合做"量大价低"的辅助任务。

## 常见误区

**误区 1：量化会严重损失模型能力**

现代量化方案（GPTQ、AWQ、Q4_K_M）在大多数通用任务上的精度损失 < 2%，用户基本感知不到差异。

**误区 2：蒸馏出来的小模型和原始小模型没区别**

通过蒸馏（尤其是过程蒸馏）训出来的小模型，在特定能力上能远超同参数规模的普通训练模型。DeepSeek R1 蒸馏版就是最好的例子。

**误区 3：参数越小越快**

推理速度不只取决于参数量，还取决于量化精度、批处理方式、硬件和推理框架。同样是 7B 模型，不同框架下速度可以差 5-10 倍。
