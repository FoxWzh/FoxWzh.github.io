> Midjourney 从来不是要做最听话的图像生成工具，v7 把这个策略推得更彻底——它宁可让图像好看，也不愿意让它完全按你说的生成。

---

## 是什么

2025 年 6 月，Midjourney v7 正式成为默认版本，取代了此前的 v6.1。如果你现在打开 Midjourney 没有指定版本，用的就是 v7。

在理解 v7 的具体功能之前，有一件事必须先说清楚：**Midjourney 的产品哲学和其他图像生成工具不一样。**

大多数图像生成工具追求的是"指令遵循"——你说什么，它生成什么，越准越好。Midjourney 的策略是**审美优先**——它有自己的审美判断，会在"按你说的做"和"生成一张好看的图"之间倾向后者。

这不是缺陷，是设计选择。但如果你拿着 Midjourney 去做需要精确控制的商业设计，就会频繁撞墙。

---

## 为什么重要

**场景一：创意团队的头脑风暴工具**

你在做产品设计的早期阶段，需要快速生成大量视觉概念来对齐团队认知。这个场景不需要"精确按 prompt 生成"，需要的是"每次都能生成一张有氛围、有质感的图"。Midjourney 在这里几乎无可替代。

**场景二：品牌视觉探索**

新品牌在定调阶段，设计师需要探索不同风格方向。Midjourney 的审美输出质量高，即使 prompt 写得比较简单，也能生成有调性的图像。这降低了"表达要求"，让非专业用户也能参与视觉探讨。

**场景三：错误选型的代价**

你的运营团队想用 Midjourney 批量生成电商产品图，要求背景是纯白、主体居中、特定光源角度。然后你会发现 Midjourney 会给你加各种它认为"好看"的元素，背景不会是你要的那种纯白，主体位置也未必居中。这种场景用错了工具，会浪费大量时间在"和模型博弈"上。

## 核心内容

### v7 的主要新特性

**RealFace 引擎**

v7 对人脸的渲染做了专项优化，目标是消除此前版本的"塑料感"。具体来说，毛孔、细纹、眼睛的水分感、皮肤的次表面散射，这些以前需要专门强调才会出现的细节，在 v7 里成了默认输出的一部分。

对于需要生成人像内容的场景，v7 的人脸质量是一个实质性提升。

**Omni Reference（--oref）**

这是 v7 里对 PM 最有产品价值的功能之一。--oref 允许你提供一张参考图，然后让新生成的图像保持和参考图一致的角色、物体或光影风格。

实际用处：你有一个品牌角色（IP 形象），想生成这个角色在不同场景下的图像，以前每次都要从头描述，效果还不稳定。--oref 让你可以用已有的图像作为"锚点"，新生成的图像在保持场景变化的同时维持主体一致性。

**风格个性化**

Midjourney 有一个持久化的个性化系统：你对生成结果进行打分（喜欢/不喜欢），系统会逐渐建立你的审美档案。开启个性化后，生成结果会往你历史喜好的方向偏移。

对于重度用户来说，这意味着用得越多、结果越"对味"。但这也意味着：你无法把自己的个性化档案迁移给别人，团队成员的生成结果会有差异。

**Draft Mode（--draft）**

速度是 Midjourney 的一个历史痛点，生成一张图要等好几秒甚至更长。Draft Mode 用 50% 的算力消耗换来约 10 倍的速度，专门用于快速预览。

实际工作流：先用 --draft 高速生成十几个方向，选中满意的构图后，再用标准模式精细生成。这个思路很实用，尤其是在创意探索阶段。

**语音输入**

v7 支持语音描述 prompt，系统会自动转写并处理。对于不擅长用文字描述视觉概念的用户，这个功能降低了使用门槛，但实际上不改变 prompt 质量问题的本质。

### 视频和 3D

Midjourney v7 同期推出了 V1 视频模型，可以把静态图转成 5 秒的短视频，并支持延伸到 21 秒。对标的是 Runway、Kling 这类视频生成产品。

3D 场景生成类似 NeRF 的效果——可以对同一场景进行多角度渲染。这个功能在产品设计原型展示、虚拟空间预览等场景有实际用处。

### 新参数一览

v7 新增了几个值得关注的参数：
- **--q 4**：高质量设置，花更多算力换更好的细节
- **--lut**：色彩分级参数，可以给图像套指定的色调风格
- **--lens**：模拟不同焦距的摄影效果（广角、长焦等）

## 能力边界/局限

**文字渲染是明显短板**

如果你要在图像里生成准确的文字，Midjourney v7 不是好选择。FLUX.1 和 Ideogram 在这一点上明显优于 Midjourney。"一张写着 Happy Birthday 的蛋糕"，Midjourney 会给你一张漂亮的蛋糕，但上面的字可能认不出来。

**指令遵循不如 GPT Image**

需要精确控制构图、元素位置、特定风格的场景，GPT-4o 图像生成的遵循度更高。Midjourney 有自己的"审美滤镜"，会在生成过程中做各种"改良"，有时候是惊喜，有时候是你不要的东西。

**商业授权需要确认**

Midjourney 的订阅协议比较复杂，不同订阅等级的商业授权范围不同。基础计划生成的图像在商用上有限制，需要查清楚具体条款。如果你要把生成内容用于商业发布，别跳过这一步。

**API 接入受限**

Midjourney 官方 API 一直不对普通开发者开放（截至本文写作时仍是企业级申请制）。大多数第三方集成走的是非官方途径，稳定性和合规性都有风险。如果你的产品需要程序化调用图像生成，Midjourney 不是可靠选项。

**使用方式限制**

主要通过 Discord 和 Web 界面使用，工作流集成不如 ComfyUI/API 方案灵活。批量生成、自动化流程在 Midjourney 上比较麻烦。

## 2026 年的最新动态

截至本文写作时（2026 年 4 月），Midjourney V8.1 Alpha 已在测试中，据悉比 v7 快 4-5 倍，同时在质量上进一步提升。正式版发布时间未定，但方向是继续推进速度和质量的同步优化。

## PM 视角的关键结论

1. **Midjourney 的核心价值是"审美输出的稳定性"**，不是"指令遵循的精确性"。用对了场景，它是创意工作中效率最高的工具之一；用错了场景，你会在 prompt 调试上浪费大量时间。

2. **--oref 是 v7 最值得关注的功能**，对于有 IP 形象一致性需求的产品，这个能力改变了工作流。

3. **商业授权和 API 接入是两个需要提前确认的硬约束**，不要到了产品上线前才发现踩了坑。

4. **Draft Mode 改变了工作节奏**。高速预览 + 精细生成的两阶段工作流，值得在团队里推广。

5. **把 Midjourney 当创意探索工具，而不是批量生产工具**。前者它做得很好，后者它不适合。

## 常见误区

**"v7 比 v6.1 全面更好"**

不一定。有部分用户反映 v7 的"审美偏好"更强，有时候反而更难按 prompt 控制。如果你的需求是精确复现 prompt，v6.1 的遵循度在某些场景下反而更高。

**"个性化功能会让所有用户受益"**

个性化需要积累打分数据才能生效，新用户短期内感受不到。另外，个性化是和账号绑定的，团队共用一个账号会产生"审美混淆"。

**"视频功能可以替代专业视频生成工具"**

Midjourney 的视频功能定位是"静态图的动效延伸"，和 Runway Gen-3、Kling 这类专门的视频生成工具相比，在运镜控制、时间一致性上仍有差距。
