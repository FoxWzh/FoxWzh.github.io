**一句话摘要：** 多模态评测体系正在从"图像能看懂吗"快速进化到"视频能理解吗"，但各个基准测的维度差异极大，用错评测数据选型会导致严重踩坑。

---

## 是什么

多模态AI的评测比纯文本复杂得多，因为"理解图像"本身就是一个包含了太多子能力的模糊概念——看懂图里有什么人、理解图表数据、识别文档里的文字、解决需要图像辅助的数学题……这些能力对应的技术路径不一样，模型的表现也千差万别。

目前主流的多模态评测可以按维度分组：

```
┌────────────────────────────────────────────────────────────┐
│ 评测基准         │ 发布  │ 核心能力           │ 难度   │
├────────────────────────────────────────────────────────────┤
│ MMBench          │ 2023  │ 图像综合理解        │ 中等   │
│ MMMU             │ 2023  │ 学科级多模态推理    │ 高     │
│ MathVista        │ 2023  │ 视觉数学推理        │ 高     │
│ DocVQA           │ 2019  │ 文档图片问答        │ 中     │
│ ChartQA          │ 2022  │ 图表理解            │ 中高   │
│ VideoMME         │ 2024  │ 视频综合理解        │ 中高   │
│ EvalCrafter      │ 2024  │ 视频生成质量        │ 评估型  │
│ MME-RealWorld-CN │ 2024  │ 中文真实场景多模态  │ 中高   │
└────────────────────────────────────────────────────────────┘
```

---

## 为什么重要

**场景一：你在做企业文档处理产品**

合同审查、财务报表分析、扫描件识别——这类产品对DocVQA和ChartQA的相关性远高于MMBench综合分。选型时看错了基准，可能选了一个"看图聊天很厉害"但"文档OCR一塌糊涂"的模型。

**场景二：你在构建教育类AI产品**

数学题拍照解答、物理实验图理解——MMMU和MathVista才是你应该重点关注的。目前顶模在MMMU上也只有65%左右的正确率，这意味着高中以上难度的题目仍有较高出错率，你需要在产品里做相应的置信度提示。

**场景三：你在做短视频或直播类AI分析产品**

如果需要模型理解视频内容（生成摘要、检测关键事件），VideoMME是最直接的参考。但要注意VideoMME测的是"视频理解"，而EvalCrafter测的是"生成视频的质量"，两个完全不同的方向，经常被混淆。

## 核心内容

### MMBench（2023，上海AI实验室）

这是当前覆盖维度最广的图像理解综合评测，包含近3000道题，覆盖物体识别、空间推理、属性判断、常识理解等二十多个子类别。当前顶模综合分在83%到89%之间。

```
顶模在MMBench各类别表现（估算）：
物体识别      ████████████████████  ~92%
属性推理      █████████████████     ~82%
空间关系      ████████████████      ~78%
图表图形理解  ██████████████        ~71%
```

MMBench适合作为图像理解的"入门筛查"，分低的模型基本可以排除，但分数接近的模型需要看垂直类别细分。

### MMMU（大学级多模态理解，2023）

这是目前难度最高的图像评测之一。题目模拟大学学科考试，涵盖医学、法律、工程、艺术史等专业领域，需要模型真正"理解"图像内容并结合学科知识推理。

顶模表现：GPT-4o约63%，Claude 3.7约65%，Gemini 1.5 Pro约60%。

这个分数说明什么？说明即便是最强的多模态模型，面对专业学科的视觉理解题，仍然有35%以上的错误率。如果你的产品涉及医疗影像分析、工程图纸理解等专业领域，这个数字意味着你不能无条件相信模型输出。

### MathVista（数学+视觉，2023）

几何题、数据图表数学题、需要视觉信息辅助的数学推理。这类任务要求模型既能"看"又能"算"，两个能力的组合比单独任何一个都难得多。

顶模表现：GPT-4o约70%，Claude 3.7约72%，Gemini 2.0约73%。人类专家在这个评测上约60%——顶模已经超过人类平均水平，但专家级人类约80%。

### DocVQA & ChartQA

DocVQA专门测文档图片（扫描PDF、截图）中的信息提取。ChartQA测柱状图、折线图、饼图的数据读取和推理。

顶模在DocVQA上可以达到92%以上，在ChartQA上约85%。但这是针对相对规范排版的图表，对于企业里那些五花八门的历史遗留文档格式，实际表现会下滑。

### VideoMME（2024，最重要的视频理解基准）

VideoMME把视频按时长分成三档：

```
┌─────────────────────────────────────────────────────────┐
│ 时长类别  │ 时长范围    │ 顶模表现  │ 主要挑战           │
├─────────────────────────────────────────────────────────┤
│ Short     │ <2分钟      │ ~72%     │ 快速动作理解        │
│ Medium    │ 2-15分钟    │ ~65%     │ 多场景连续理解      │
│ Long      │ >15分钟     │ ~58%     │ 长程时间推理        │
└─────────────────────────────────────────────────────────┘
```

随着视频变长，所有模型的表现都会显著下降。这是目前多模态的一个核心瓶颈——长视频理解需要在有限的context window里处理大量时序信息。

**2025年当前顶模横向对比：**

```
┌──────────────────────────────────────────────────────────────┐
│ 评测         │ GPT-4o  │ Claude 3.7 │ Gemini 2.0 Pro        │
├──────────────────────────────────────────────────────────────┤
│ MMBench      │  87%    │  88%       │  86%                  │
│ MMMU         │  63%    │  65%       │  62%                  │
│ MathVista    │  70%    │  72%       │  73%                  │
│ DocVQA       │  92%    │  91%       │  90%                  │
│ ChartQA      │  85%    │  84%       │  83%                  │
│ VideoMME(平均)│  64%   │  62%       │  68%                  │
└──────────────────────────────────────────────────────────────┘
（数据为2025年初估算）
```

有意思的地方在于：Gemini在VideoMME上的表现明显优于对手，这和Google在视频处理上的技术积累有关；Claude在MMMU和MathVista上表现最好，这和Anthropic对推理能力的着重投入有关。

### 中文多模态：MME-RealWorld-CN

国内团队发布的中文真实场景多模态评测，包含中文路牌识别、中文文档理解、中文图表分析等任务。在这个评测上，国产模型（如Qwen-VL-Plus、InternVL2）的表现普遍比国际模型好10-20个百分点，尤其在中文OCR和中文图表理解上。

这个信息对做中文市场产品的PM很重要：涉及中文文字内容的多模态任务，国产模型值得认真考虑，而不是默认用GPT-4o。

## 能力边界与局限

**当前评测体系的共同局限：**

多模态评测几乎都是选择题格式，用准确率衡量。但真实产品里，模型的输出是开放式的，如何评估"描述图片内容"的质量，这些评测并没有很好的答案。

**视频评测的特殊困境：**

VideoMME测的是视频理解，但当前很多模型处理视频的方式其实是"抽帧然后逐帧理解"，并非真正的时序建模。抽帧策略（每秒抽几帧、关键帧选择）对结果影响很大，但这是实现细节，评测不区分。

**基准污染风险：**

MMMU和MathVista的题目已经广泛流传，部分模型训练数据可能包含这些题目的答案。分数高可能是"见过"，不是"真的会"。

## PM视角关键结论

1. **按产品功能对应具体评测，而不是看综合分**。产品是文档处理：重点看DocVQA。产品是教育数学辅导：重点看MathVista和MMMU。产品是视频分析：看VideoMME且要关注你实际视频时长对应的子类别。

2. **中文场景要专门测中文多模态能力**。不要用英文评测结果推断中文表现。MME-RealWorld-CN的数据显示，国际模型在中文视觉任务上表现往往差国产模型15-20%。

3. **视频理解能力当前还在爬坡阶段，产品设计要留余量**。VideoMME顶模长视频才58%，意味着约4成的长视频问题会答错。如果你的产品依赖模型准确理解长视频，要在产品设计上加入人工审核环节。

4. **MathVista分数对教育类产品是强选型指标**。数学+视觉是教育AI的核心能力，各模型在MathVista上只有3-5%的差距，但在具体数学子类别上差异可能超过10%。

5. **当心"视频理解"和"视频生成质量"的概念混淆**。EvalCrafter、VBench测的是"文生视频"的质量，VideoMME测的是"视频内容理解"。做视频内容分析产品，参考VideoMME；做AI视频生成产品，参考EvalCrafter。混用评测数据做选型决策是一个行业里比较常见的低级错误。

## 常见误区

**误区一：多模态综合分高就意味着OCR能力强**

这是文档处理类产品最常踩的坑。MMBench和MMMU的高分不代表模型擅长识别密集文字、复杂版式的文档。有团队基于综合分选了模型，结果发现合同扫描件的关键条款经常被漏读或误读。

**误区二：视频理解能力等于看多少帧**

有产品团队觉得，只要增加抽帧数量，模型的视频理解能力就会提升。但实验发现，从每秒1帧增加到每秒4帧，在VideoMME上的提升远小于预期，还大幅增加了成本。真正的视频理解瓶颈在于模型的时序推理能力，不只是"看到"更多帧。

**误区三：评测分数一旦高于人类就代表可以完全替代人工**

MMMU有些子类别顶模已经接近或超过"平均大学生"水平，但"平均大学生"和"专业审核员"之间还有巨大差距。而且模型出错的方式和人类出错的方式不同——模型会在奇怪的地方确定性地犯错。这种系统性错误模式在高风险场景下是危险的。
