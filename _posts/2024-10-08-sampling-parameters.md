> 一句话摘要：这三个参数是控制 AI "说话风格"的旋钮——读完你能在写 Prompt 或配置 API 时，知道该拧哪个旋钮、拧多大。

---

## 是什么

大模型每次生成一个字，本质上都是在做一道"概率选择题"：它给词库里几万个词每人打一个概率分，然后按某种规则从中挑一个。

如果永远只选概率最高的那个词，模型就会变得死板、重复——就像打字法输入法永远给你推荐同一个词一样。**采样参数就是为了打破这种机械感，让模型在"准确"和"有创意"之间找到平衡。**

三个核心参数：

- **Temperature**：控制模型有多"大胆"
- **Top-K**：限定候选词的最多数量
- **Top-P**：按累积概率动态圈定候选词

---

## 为什么重要

### 场景 1：客服机器人总是答非所问

配置了太高的 Temperature，模型开始"发挥"，明明是问退款流程，它聊起了品牌故事。调低 Temperature，问题立刻消失。

### 场景 2：AI 写作工具写出来的内容千篇一律

Temperature 调得太低，所有用户生成的文章都像一个模子刻出来的，用户投诉"没有个性"。适当调高，内容立刻丰富起来。

### 场景 3：API 调用结果不稳定，同样的 Prompt 每次输出差很多

Top-P 和 Temperature 都没有设定，用了模型默认值，导致输出随机性过大。锁定参数后，输出稳定可控。

---

## 核心内容

### 先理解模型的"选词过程"

每次生成一个 token，模型内部大概是这样的：

```
词库（10万个词）
→ 每个词获得一个概率分
→ 采样参数过滤 + 调整
→ 从候选池中随机抽取一个
→ 输出这个词
→ 重复以上过程
```

没有采样参数的话，模型永远选概率第一的词，输出必然是固定的。加了采样参数，才有了随机性和多样性。

---

### Temperature：创意旋钮

**直觉理解**：Temperature 越高，模型越"大胆"，越愿意选那些概率排名靠后的词。

数学上，它是用来缩放概率分布的：低温度让高概率词更突出，高温度让概率分布趋于平均。

```
Temperature = 0.1（低温，保守）
  苹果: 95%  西瓜: 4%  香蕉: 1%  ...
  → 几乎必选"苹果"

Temperature = 1.5（高温，大胆）
  苹果: 40%  西瓜: 35%  香蕉: 15%  ...
  → 三者都有机会被选到
```

**实用区间：**

| 范围 | 表现 | 适用场景 |
|------|------|---------|
| 0 ~ 0.3 | 极度稳定，几乎无随机性 | 结构化数据提取、代码生成、事实问答 |
| 0.4 ~ 0.7 | 自然流畅，有轻微变化 | 通用对话助手、文档摘要 |
| 0.8 ~ 1.0 | 有创意，偶有惊喜 | 营销文案、邮件撰写 |
| 1.0 以上 | 发散、有时离谱 | 头脑风暴、创意写作（需人工筛选） |

> **PM 注意**：Temperature > 1.5 基本意味着输出不可控，生产环境慎用。

---

### Top-K：设定候选人数量上限

**直觉理解**：只从概率最高的 K 个词里选，其余全部排除在外。

```
Top-K = 5
词库中概率排名：苹果(40%) 西瓜(30%) 香蕉(15%) 草莓(10%) 葡萄(3%) 核潜艇(0.001%)...
→ 只保留前5个，"核潜艇"这种离谱词永远不会出现
```

**作用**：防止模型偶尔抽到极低概率的"离谱词"，是一道安全防线。

**典型设置**：40 ~ 100。设得太小（如 K=3），模型输出会变得单调；设得太大，等于没过滤。

---

### Top-P（Nucleus Sampling）：动态圈定候选池

**直觉理解**：不限定数量，而是按"靠谱程度总和"来圈定候选范围——把词按概率从高到低排列，一直加到总和达到 P 为止。

```
Top-P = 0.9

情况A：模型非常确定
  "白日依山..."后面
  尽: 88%  完: 5%  了: 3%  ...
  → 只需2个词就达到90%，候选池极小 → 输出稳定

情况B：模型不确定
  "我喜欢吃..."后面
  苹果: 15%  西瓜: 12%  香蕉: 11%  ...（需要很多词才能凑到90%）
  → 候选池变大 → 输出多样
```

**为什么比 Top-K 更聪明**：Top-K 是"永远只看前50名"，不管模型确不确定。Top-P 是"模型越确定，候选池越小"，更贴近实际需要。

**典型设置**：0.85 ~ 0.95。

---

### 三者的关系和使用建议

这三个参数通常叠加生效，执行顺序一般是：**Top-K → Top-P → Temperature**。

实际工程中，主流建议是：

> **只调 Temperature + Top-P，不碰 Top-K**（或把 Top-K 设成一个很大的值等于不生效）

原因：Top-P 已经能起到 Top-K 的过滤作用，且更智能。同时调两个过滤参数容易产生意外的叠加效果，难以调试。

**各场景推荐配置：**

| 场景 | Temperature | Top-P | 说明 |
|------|-------------|-------|------|
| JSON/结构化输出 | 0 ~ 0.1 | 1.0 | 追求完全一致，不要任何随机性 |
| 代码生成 | 0.2 | 0.9 | 精确为主，略有灵活性 |
| 通用对话助手 | 0.7 | 0.9 | 大多数产品的默认值 |
| 营销文案 | 0.8 | 0.9 | 有创意但不失控 |
| 创意写作 | 1.0 ~ 1.2 | 0.95 | 允许有惊喜 |

---

### 2024 年新趋势：Min-P

在开源模型社区（Llama 3、Mistral 等），出现了一个新参数 **Min-P**：

> 以概率最高的词为基准，只保留概率至少是它 P 倍的词。

比如最高概率是 80%，Min-P = 0.05，则只保留概率 ≥ 4% 的词。

**PM 层面的意义**：比 Top-P 更动态，在创意写作场景下能更好地防止输出"崩掉"。如果你的产品基于开源模型，工程师可能会提到这个参数。

## 能力边界 / 局限

- **这些参数不能修复训练数据的问题**。Temperature 再低，如果模型本身对某个事实就是错的，输出还是错的。
- **Temperature = 0 也不是绝对确定性**。在某些实现里，温度为 0 仍然有极小的随机性（取决于硬件浮点精度）。真正要求可复现，需要同时固定 `seed`。
- **参数效果因模型而异**。同样是 Temperature = 0.7，GPT-4o 和 Llama 3 的实际输出多样性可能差别很大，因为模型的概率分布本来就不同。
- **不能解决幻觉问题**。低 Temperature 能减少"乱说"，但无法根本上消除幻觉，那是训练层面的问题。

---

## PM 视角的关键结论

1. **Temperature 是你最常需要调的参数**。大多数场景只需要调它，Top-P 保持默认（0.9）即可。

2. **结构化输出场景，Temperature 直接设 0**。JSON 提取、表格填充、分类打标——不需要任何随机性，设 0 是最安全的做法。

3. **不要同时大幅改动多个参数**。一次只改一个，观察效果再调下一个，否则根本搞不清楚是哪个参数在起作用。

4. **生产环境要锁定参数，并写进文档**。不同环境用不同参数会导致输出不一致，这是很多线上问题的隐性根因。

5. **Temperature 高不代表创意好**。超过 1.2 之后，增加的主要是"噪音"而不是"创意"——需要人工筛选才有价值。

---

## 常见误区

**误区 1：Temperature 越高，模型越聪明**

Temperature 只影响随机性，不影响模型的知识储备和推理能力。高 Temperature 只是让模型更愿意选"冷门词"，跟聪不聪明没关系。

**误区 2：Top-P 和 Top-K 要同时设置才有效**

实际上两者功能有重叠，同时设置小值反而会产生意外的叠加限制。主流实践是选一个用，通常选 Top-P。

**误区 3：调参可以替代 Prompt 优化**

采样参数控制的是"怎么选词"，而不是"模型理解了什么"。Prompt 写得不清楚，把 Temperature 调到 0 也没用——模型还是会按它理解的方向输出。
